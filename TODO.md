## 已支持的文件类型

- 音频预览（mp3, wav, ogg, m4a, flac, aac）
- PDF 预览（使用 PDF.js Viewer）
- 视频预览（mp4, webm, mov, avi, mkv, ogg）— 支持分块流式加载和网络感知警告

## 候选文件类型

GitHub 自身不支持原生预览、但对开发者有实用价值的文件类型：

| 类型          | 扩展名                    | 实现成本    | 实用价值  | 与现有架构契合度                   |
| ------------- | ------------------------- | ----------- | --------- | ---------------------------------- |
| 视频          | .mp4, .webm, .mov, .avi   | ⭐ 低       | ⭐⭐⭐ 高 | 与音频几乎一致，复用所有 UI 和流程 |
| 压缩包浏览器  | .zip, .tar.gz, .7z, .rar  | ⭐⭐ 中     | ⭐⭐⭐ 高 | 纯 JS 解压库（JSZip），新 UI       |
| Excel 表格    | .xlsx, .xls               | ⭐⭐⭐ 中   | ⭐⭐⭐ 高 | 需要 SheetJS 库                    |
| Word 文档     | .docx                     | ⭐⭐ 低     | ⭐⭐ 中   | 可用 mammoth.js 解析               |
| SQLite 查看器 | .sqlite, .db, .sqlite3    | ⭐⭐⭐⭐ 高 | ⭐⭐ 中   | 需要 sql.js (WASM)，体积较大       |
| HEX 查看器    | .bin, .hex, .eeprom       | ⭐⭐ 中     | ⭐⭐ 中   | 二进制/固件文件查看                |
| 字体预览      | .ttf, .otf, .woff, .woff2 | ⭐⭐ 中     | ⭐ 一般   | 在网页上渲染字体预览               |

## 推荐优先级

1. **视频** — 实现成本最低，与现有音频架构几乎完全一致
2. **压缩包浏览器** — 纯前端方案，无需后端
3. **Excel 表格** — 实用价值高，常见于配置/数据文件
4. **Word 文档** — 实现成本低，但使用场景相对少
5. **SQLite 查看器 / HEX 查看器 / 字体预览** — 依赖更重的库，可延后
